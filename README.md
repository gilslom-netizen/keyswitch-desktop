# KeySwitch Desktop

תוכנת Windows לזיהוי ותיקון אוטומטי של טעויות שפת מקלדת עברית/אנגלית — **בכל תוכנה במחשב**, לא רק בדפדפן.

הגרסה הזו היא האחות של [תוסף הכרום KeySwitch](https://chromewebstore.google.com/detail/keyswitch/cgnejgnihpbiphadifipmoimdaaechka), עם אותו מנוע זיהוי, אותם מילונים ואותו עיצוב — אבל עם יכולות שתוסף דפדפן לא יכול לספק.

## מה התוכנה עושה

- **תיקון אוטומטי בזמן הקלדה** — מזהה מילים שנכתבו בפריסת מקלדת שגויה ומתקנת אותן תוך כדי כתיבה, בכל מקום: Word, Excel, דפים חסומים של כרום, Google Docs ודפי קנבס, טרמינל — כל מקום שמקלידים בו.
- **החלפת שפת מקלדת אמיתית** — במקום להמיר כל תו שמוקלד לתו המקביל (כמו בתוסף), התוכנה מתקנת את הטקסט השגוי ופשוט **מחליפה את שפת המקלדת של Windows**, כך שההקלדה ממשיכה טבעית בשפה הנכונה.
- **טיפול חכם ב-CapsLock** — אם התוכנה מזהתה שהטעות נגרמה מ-CapsLock דלוק, היא קודם מכבה את ה-CapsLock, ואז בודקת האם שפת המקלדת כבר נכונה — ומחליפה אותה רק אם צריך. תיקון הטקסט עצמו נעשה בהזרקת Unicode שאינה תלויה בפריסה, כך שהוא עובד גם במקרה זה.
- **המרה ידנית בקיצור מקלדת** — סמנו טקסט בכל תוכנה ולחצו `Alt+Shift+J` (ניתן לשינוי בהגדרות). הטקסט מוחלף ושפת המקלדת מתיישרת לשפת התוצאה.
- **נגישה מהחיפוש של Windows** — האינסטלר יוצר קיצור בתפריט התחל ובשולחן העבודה; מקלידים "KeySwitch" בחיפוש של Windows ומגיעים ישר לחלון ההגדרות. אין צורך לחפש קבצים בסייר.
- **אותו מראה של התוסף** — חלון ההגדרות, ההודעות הצפות (Toast) והממשק כולו זהים בעיצובם לפופאפ של התוסף, כולל גופן WixMadeforText וערכת הצבעים הכהה.
- **כפתור "החזר טקסט מקורי ועצור"** — כמו בתוסף: כל תיקון אוטומטי מציג הודעה עם אפשרות ביטול מלאה (מחזירה גם את הטקסט, גם את שפת המקלדת וגם את מצב ה-CapsLock), עם מנגנון למידה שמפסיק לתקן מילים שדחיתם.

## התקנה

מורידים את `KeySwitch-Setup-<version>.exe` מעמוד ה-[Releases](../../releases) ומריצים. האינסטלר הוא אשף גרפי מקצועי (NSIS) — לא חלון CMD — הכולל **עמוד הגדרות** עם בדיוק אותן אפשרויות שיש בתוסף:

- תיקון אוטומטי בזמן הקלדה
- הצגת הודעה לאחר תיקון אוטומטי
- הצגת הודעה לאחר המרה ידנית
- שפת הקלדה עיקרית (עברית / חצי חצי / אנגלית)
- הפעלה אוטומטית עם Windows

ההגדרות נכתבות ל-`%APPDATA%\KeySwitch\settings.json` — אותו קובץ שהתוכנה קוראת. בעדכון גרסה (כשכבר קיימות הגדרות) עמוד ההגדרות מדלג אוטומטית ולא דורס את ההעדפות הקיימות.

## הטמעת הגדרות בקישור ההורדה (KSCFG)

אפשר להגיש למשתמש אינסטלר שכבר ממולא מראש בהגדרות שבחר — למשל בהפניה מהתוסף. מוסיפים לשם קובץ האינסטלר (או לשורת הפקודה) אסימון `KSCFG` בן 5 תווים:

```
KSCFG<a><b><c><d><e>
```

| מיקום | משמעות | ערכים |
|---|---|---|
| 1 | תיקון אוטומטי בזמן הקלדה | `0`/`1` |
| 2 | הודעה לאחר תיקון אוטומטי | `0`/`1` |
| 3 | הודעה לאחר המרה ידנית | `0`/`1` |
| 4 | הפעלה עם Windows | `0`/`1` |
| 5 | שפת הקלדה עיקרית | `h`=עברית, `e`=אנגלית, `m`=חצי חצי |

דוגמאות:

```
KeySwitch-Setup-1.0.0-KSCFG1111h.exe        ← הכל פעיל, עברית עיקרית
KeySwitch-Setup-1.0.0-KSCFG1011e.exe        ← בלי הודעת תיקון אוטומטי, אנגלית עיקרית
KeySwitch-Setup.exe /KSCFG=1101m            ← דרך שורת הפקודה
```

שרת ההורדה של התוסף יכול פשוט לשנות את שם הקובץ המוגש (או להגדיר `Content-Disposition: attachment; filename=...KSCFG1111h.exe`) לפי ההגדרות הנוכחיות של המשתמש בתוסף — עמוד ההגדרות באינסטלר ייפתח ממולא מראש בהתאם. התקנה שקטה (`/S`) עם אסימון תכתוב את ההגדרות ישירות.

## איך זה עובד (ארכיטקטורה)

| רכיב | תפקיד |
|---|---|
| `src/engine/shared_logic.js` | מיפוי הפריסות + `convertFullText()` — **זהה בייט-בייט לתוסף** (עם תוספת export) |
| `src/engine/dictionaries.js` | המילונים וסיווג מילים (`classify`) — פורט מ-autocorrect.js של התוסף |
| `src/engine/keymap.js` | תרגום מקשים פיזיים (keycodes) לתו שמופיע על המסך לפי פריסה, Shift ו-CapsLock |
| `src/engine/autocorrect-engine.js` | המנוע: בניית "ריצת הקלדה" מה-hook הגלובלי, זיהוי, תיקון, החלפת שפה, ביטול |
| `src/native/win.js` | שכבת Win32 (koffi): החלפת פריסה, CapsLock, הזרקת מקשים (SendInput) |
| `src/main.js` | Electron: מגש מערכת, חלון הגדרות, Toast, קיצור גלובלי, המרה ידנית |
| `build/installer.nsh` | עמוד ההגדרות של האינסטלר + פענוח KSCFG + כתיבת settings.json |

ההבדל המהותי מהתוסף: התוסף נכנס ל"סשן חי" שממפה כל הקשה עוקבת לתו המקביל; התוכנה מחליפה את פריסת המקלדת של המערכת (`WM_INPUTLANGCHANGEREQUEST`) כך שההקשות הבאות פשוט נכונות מלכתחילה. תיקון הטקסט שכבר הוקלד נעשה עם Backspace סינתטי והקלדה מחדש ב-`KEYEVENTF_UNICODE` — בלתי תלוי בפריסה ועובד בכל אפליקציה.

## פיתוח

```bash
npm install          # התקנת תלויות
npm test             # בדיקות המנוע (רץ בכל מערכת הפעלה)
npm run lint         # בדיקת תחביר
npm start            # הרצת האפליקציה (Windows בלבד לפונקציונליות מלאה)
npm run dist         # בניית אינסטלר NSIS (על Windows)
```

ה-CI (GitHub Actions) מריץ את הבדיקות על Ubuntu ובונה את האינסטלר על Windows; דחיפת תג `v*` יוצרת Release עם קובץ ההתקנה.

## מגבלות ידועות

- Windows בלבד (החלפת פריסה, hook גלובלי ו-SendInput הם Win32).
- התיקון האוטומטי מוחל בלחיצה על רווח. Enter ו-Tab לא מפעילים תיקון (בכוונה — Enter עלול לשלוח הודעה בצ'אטים לפני שאפשר לתקן).
- ההמרה הידנית משתמשת בלוח (Clipboard) ומשחזרת את תוכנו הטקסטואלי לאחר ההדבקה.
